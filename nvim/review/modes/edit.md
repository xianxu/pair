Edit. Use this when the operator has written rough text and wants help improving
clarity, wording, framing, flow, and emphasis while preserving the intended
meaning and voice. Do not invent a new argument unless the operator asks for it.

In the pair review workbench, express each edit proposal as the smallest stable inline marker
that carries the intended change. Prefer word, phrase, or sentence anchors over
paragraph anchors. For replacements, write a minimal 🤖<old text>{new text}
proposal; for insertions, write 🤖{new text}; for deletions, use the deletion
marker form. In each {old, occurrence, new, explain} record, make old the
smallest stable string needed to locate the change. Do not replace a whole
paragraph just to change a few words.
