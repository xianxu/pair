# create visualization of change log in a pair session

## Problem

right now, pair-slug would generate a per turn summary of what's going on, which helps operator orienting. this is a furtherance of that idea.

ideally we would have a changelog easily accessible to operator of the changes in the session. each entry in the change log doesn't necessarily perfectly align with a turn, but some notable change/decision etc. it should be concise to allow quick glance, but longer than the slug format generated by pair-slug.

1/ we can bind alt+l to it (log).
2/ and it would open an nvim window, full screen to display such change log.
3/ this nvim is readonly, like the scrollback.
4/ I expect it has key milestones, such as "start M1", "M1 done", etc. as well as other key decisions: "brainstorm on feature, decided ...".
