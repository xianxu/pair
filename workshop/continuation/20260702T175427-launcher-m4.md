### M4 design (implement this)

**The cutover is: flip the default, NOT retire the shell.** Today the gate is
`if os.Getenv("PAIR_NATIVE_LAUNCH") != "" { LaunchNative(...); if !ErrFallbackToShell
return }` then exec `bin/pair-shell` (cmd/pair-go/main.go:130-137). M4 makes the
native launcher run **by default** (native-first), with `bin/pair-shell` as the
fallback for the still-`ErrFallbackToShell` surfaces.

- **Flip the gate** so native runs unless explicitly opted OUT (e.g. invert to a
  `PAIR_LEGACY_LAUNCH`/`PAIR_NO_NATIVE_LAUNCH` escape hatch, or default-on with an
  env kill-switch for safety during rollout). Native-first; `ErrFallbackToShell`
  still execs the **real** `bin/pair-shell`.
- **⚠️ CRUX — do NOT convert `bin/pair-shell` to a thin shim → `pair-go launch` in
  M4.** The native launcher still returns `ErrFallbackToShell` (pick / compaction /
  rename / continue). If the fallback target were a shim back to `pair-go launch`,
  that would re-enter native → `ErrFallbackToShell` → shim → **infinite loop**. So
  `bin/pair-shell` MUST stay the real launcher for the fallback paths until **M5**
  makes them native. The plan's M4 bullet says "convert bin/pair-shell to a thin
  shim" — that half is really **M5** (retirement); narrow the M4 bullet at close
  (Revision + narrowed tick — see lessons.md "milestone that defers scope"). M4 =
  flip the default + prove e2e; shell stays the real fallback.

## ⚠️ sdlc milestone-close window bug (ariadne#162, still open)
`sdlc milestone-close`'s auto-windows are buggy: the review window can pick a
far-back base → huge diff → `fork/exec claude: argument list too long`; the atlas
window can be `base=HEAD` → empty → false "no atlas changes". **Workaround (used
for M1/M2/M3):** run the review manually (`sdlc judge milestone-review --base
"$(git merge-base main HEAD)" --head HEAD --issue 99`), then `sdlc milestone-close
… --no-judge` (add `--no-atlas` ONLY if the atlas window is genuinely empty, naming
the atlas commit in `--verified`). For M3 the prev-boundary base was correct (the
window was small) so the atlas gate passed without `--no-atlas` — check per close.
**Put the REAL verdict in the milestone commit's `Review-Verdict:` trailer** (final
`sdlc close` greps commits for it — do NOT leave only sdlc's `not-run`). Lesson +
the window-bug variants are in `workshop/lessons.md`.
