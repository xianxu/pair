## Lessons learned
- **A modified-buffer bare `qall` hangs `nvim --headless` forever** (E37, even with
  stdin=`/dev/null`). Any headless-nvim driver that mutates a buffer must `qall!`.
  The class is latent across drivers — audit siblings, don't fix only the one that
  bit.
- **Never let a sub-process boot run unbounded in a test suite.** One stuck child
  hung `make test` 12m54s and leaked week-old corpses. A timeout watchdog turns an
  infinite hang into a bounded, loud `124` — and `>/dev/null 2>&1 || true` was
  actively hiding both the hang and the diagnostic.
- **When a fix removes the only trigger of a safety path, pin that path with a
  fixture** — else a green suite ships the contract unproven.
- **Reproduce streaming, not through `tail`.** `… 2>&1 | tail` buffers until EOF, so
  a *progressing* run looks frozen — the original "hang" report was partly this
  measurement artifact layered on a real stall. Stream output when timing a suspected
  hang.
