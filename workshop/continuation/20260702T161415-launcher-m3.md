## ⚠️ sdlc milestone-close window bug (BOTH variants — filed as ariadne #162)
`sdlc milestone-close`'s auto-windows are buggy: (1) the **review** window can pick
a far-back base → 6.8 MB diff → `fork/exec claude: argument list too long`; (2) the
**atlas gate** window can be `base=HEAD` → empty → false "no atlas/ changes".
**Workaround:** run the review manually (`sdlc judge milestone-review --base
"$(git merge-base main HEAD)" --head HEAD --issue 99`), fix findings, then
`sdlc milestone-close … --no-judge` (and `--no-atlas` if the atlas-window is empty,
naming the atlas commit in `--verified`). Put the REAL verdict in the milestone
commit's `Review-Verdict:` trailer (final `sdlc close` greps commits for it — do
NOT leave only sdlc's `not-run`). Filed upstream as **ariadne #162** this session;
lesson recorded in `workshop/lessons.md`.
