## Lessons learned

- **Fix the durable plan, not just the continuation.** A scope contradiction left
  in the plan is a change-code plan-quality FAILURE even when you fully understand
  it and explained it in the continuation — the judge reads the plan, and an agent
  reading only the plan would hit the bug. (Pensive + lessons.md.)
- **`sdlc milestone-close --actual` is issue-cumulative, not the increment.**
  Compute `cumulative_now − cumulative_at_prior_close` for the re-price note; pass
  the tool's cumulative value to `--actual` (measured, not typed).
- **A native-default cutover surfaces latent parser gaps** (the `--help`-as-agent
  bug was invisible under the opt-in flag). When flipping a default, e2e the
  *fallback* surfaces, not just the happy path — the M4 smoke asserting
  "--help → shell exactly once (no loop)" caught the class the plan judge named.
- **Milestone-close window bug (ariadne#162) persists** — run the review manually
  (`sdlc judge milestone-review --base <merge-base>`), `--no-judge` the close, put
  the REAL verdict in the `Review-Verdict:` trailer (final `sdlc close` greps for
  it). The review judge is a ~5–8min LLM pass — background it, don't foreground a
  2min timeout.
